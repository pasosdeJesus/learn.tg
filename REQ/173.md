# R-#173: "Partial login" after navigating away from landing page

## Symptom
After connecting wallet on `https://learn.tg/`, clicking English shows "Partial
login" on `/en`. Also happens navigating to Profile or any other page.

## Root Cause

The SIWE auth system is designed around every page load triggering a new
wallet connection → SIWE sign-in → token rotation (doc/siwe-auth-flow.md:93).

R-#167 added `reconnectOnMount={false}` which prevented wagmi from
auto-reconnecting on page mount. Without reconnection, SIWE never fires, the
DB token stays stale, and API calls fail → "Partial login".

Additional contributor: `<a href="/en">` (plain HTML anchor in `app/page.tsx`)
caused hard navigation, destroying the React tree. This amplified the effect
but was not the sole cause.

## Fix

- [x] Revert `reconnectOnMount={false}` — restore wagmi auto-reconnect behavior
- [x] `<a href="/en">` → `<Link href="/en">` in `app/page.tsx` — reduces remounts
- [x] `NEXTAUTH_URL` corrected in production `.env` (was localhost:3000)
- [x] Browser console diagnostics added to Partial login guards (all 4 pages)

## Verification

Deploy and test manually with OneKey wallet:
1. Open https://learn.tg → Connect Wallet → sign
2. Click English → should show courses without "Partial login"
3. Click Profile → should show profile without "Partial login"
