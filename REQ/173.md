# R-#173: "Partial login" after navigating away from landing page

## Symptom
After connecting wallet on `https://learn.tg/`, clicking English shows "Partial
login. Please disconnect your wallet and connect and sign again." on `/en`.
Also happens navigating to Profile or any other page.

## Root Cause

Confirmed via server logs + browser console (2026-07-06):

1. `app/page.tsx` uses `<a href="/en">` — a plain HTML anchor, not Next.js
   `<Link>`. This triggers a **full page reload**, destroying the React tree.
2. `AppProvider` has `reconnectOnMount={false}` (R-#167), so wagmi stays
   disconnected after remount.
3. Profile/course pages check `useAccount().address` at render time — it's
   `undefined` because wagmi never reconnected → guard fails → "Partial login".

**The auth system is NOT broken.** API token, session cookies, and SIWE all work
correctly. The page guard just can't find a wallet connection because the React
tree was destroyed by the `<a>` hard navigation.

### Evidence
- Server logs show `authenticateUser` succeeding with correct token/wallet
- Browser console shows profile API calls returning 200 with correct data
- Session cookies persist across navigation (NextAuth JWT intact)
- Only `useAccount().address` is missing after hard nav with reconnect disabled

### Also fixed
- `NEXTAUTH_URL` was `http://localhost:3000` in production `.env` — fixed
  (caused `CLIENT_FETCH_ERROR` on CSRF token endpoint)

## Fix

Already applied:
- [x] `<a href="/en">` → `<Link href="/en">` in `app/page.tsx` (same for `/es`)
- [x] `reconnectOnMount={false}` in `AppProvider.tsx`
- [x] `NEXTAUTH_URL` corrected in production `.env`
