## Phase 1: Notifications (MVP)

### 1.1 Notification Types

| Type | Trigger | Recipient | Channel |
|------|---------|-----------|---------|
| Guide completed | User completes guide | User | In-app, Email |
| Cluster milestone reached | Cluster reaches milestone | Cluster members | In-app, Email |
| Donation received | Donation to cluster | Cluster admin | In-app, Email |
| GD contact status changed | Contact status changes | Cluster admin, pdJ | In-app, Email |
| Funds released | Funds released | Cluster admin | In-app, Email |
| 15-day timeout | 15 days no GD response | Cluster admin, pdJ | In-app, Email |
| Aave yield update | Daily yield | Cluster admin | Email |

### 1.2 Notification Channels

| Channel | Priority | Description |
|---------|----------|-------------|
| **In-app** | High | Notifications in the learn.tg interface |
| **Email** | Medium | Email notifications (if email is available) |
| **WhatsApp** | Low | WhatsApp notifications (future) |

### 1.3 Notification Preferences

| Option | Description |
|--------|-------------|
| **Email notifications** | Enable/disable email notifications |
| **In-app notifications** | Enable/disable in-app notifications |
| **Notification types** | Select which types of notifications to receive |

## Phase 2: Internal Messaging (Post-MVP)

### 2.1 Messaging Types

| Type | Description | Participants |
|------|-------------|--------------|
| **Cluster messages** | Communication between churches in the same cluster | Pastors, Delegates |
| **pdJ messages** | Support and verification | Cluster ↔ pdJ |
| **Donor messages** | Thank you notes and updates | Cluster ↔ Donors |
| **GD messages** | Direct communication with GD (future) | Cluster ↔ GD |

### 2.2 Messaging Features

| Feature | Description |
|---------|-------------|
| **Threaded conversations** | Each conversation is a thread |
| **Attachments** | Attach files (documents, images) |
| **Encryption** | Messages encrypted using wallet-based encryption (#153) |
| **Read receipts** | See if messages have been read |

### 2.3 Security and Privacy

| Aspect | Implementation |
|---------|----------------|
| **Encryption** | Messages encrypted with recipient's public key |
| **Permissions** | Only cluster members can see cluster messages |
| **Anonymity** | Anonymous clusters can communicate without revealing identity |
| **Audit** | Messages with pdJ are auditable |

## 3. Database Schema

### 3.1 Notifications

```sql
CREATE TABLE notifications (
    id SERIAL PRIMARY KEY,
    usuario_id INTEGER REFERENCES usuario(id),
    type VARCHAR(50),
    title VARCHAR(200),
    content TEXT,
    link VARCHAR(500),
    read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 3.2 Messages (Phase 2)

```sql
CREATE TABLE messages (
    id SERIAL PRIMARY KEY,
    sender_id INTEGER REFERENCES usuario(id),
    recipient_id INTEGER REFERENCES usuario(id),
    cluster_id INTEGER REFERENCES cluster(id),
    subject VARCHAR(200),
    content TEXT,
    encrypted BOOLEAN DEFAULT TRUE,
    read BOOLEAN DEFAULT FALSE,
    parent_id INTEGER REFERENCES messages(id), -- for threading
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

## 4. API Endpoints

| Endpoint | Method | Description |
|----------|--------|-------------|
| `/api/notifications` | GET | Get user's notifications |
| `/api/notifications/:id/read` | PUT | Mark notification as read |
| `/api/notifications/preferences` | GET/PUT | Get/update notification preferences |
| `/api/messages` | GET | Get user's messages |
| `/api/messages` | POST | Send a message |
| `/api/messages/:id` | GET | Get a message thread |
| `/api/messages/:id/read` | PUT | Mark message as read |
