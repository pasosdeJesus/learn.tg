# R-#186: Replace RainbowKit with Own Wallet/Auth Implementation

Replace the RainbowKit → wagmi → SIWE tower with a minimal, controlled
implementation that uses NextAuth session as the single source of truth
for user identity.

## Motivation

RainbowKit's `RainbowKitSiweNextAuthProvider` couples wallet connection state
(wagmi) with authentication state (NextAuth). When wagmi's React context loses
state on client-side navigation (NextAuth bug #5719), RainbowKit interprets
this as "user needs to sign in again" and triggers the full SIWE flow with
connection popups. This produces a broken UX: "Connect Wallet" appearing
on every page, multiple "Accept connection" dialogs, and unnecessary
re-authentication.

The root cause is **not** wagmi or NextAuth individually — it's the coupling
RainbowKit creates between them. Both wagmi and NextAuth work fine in isolation.

## Design

### Current Stack (to replace)

```
RainbowKit ConnectButton
  └─ RainbowKitSiweNextAuthProvider
       ├─ wagmi (useAccount, useConnect)
       └─ NextAuth (useSession)
```

### Proposed Stack

```
Own Header component (session.address + custom Connect button)
  ├─ NextAuth (useSession)        ← identity, always available
  └─ window.ethereum (direct)     ← signing, only when needed
```

### Key Changes

#### 1. Replace `ConnectButton` with Own UI

```tsx
// Instead of RainbowKit's <ConnectButton />
// We render our own button based on NextAuth session:

{!session?.address ? (
  <button onClick={handleConnect}>Connect Wallet</button>
) : (
  <span>{session.address.slice(0,6)}...{session.address.slice(-4)}</span>
)}
```

The `handleConnect` calls `window.ethereum.request({ method: 'eth_requestAccounts' })` directly — no wagmi, no RainbowKit.

#### 2. SIWE on First Connect Only

On first connect (or when user explicitly clicks "Connect"), run SIWE:
```
eth_requestAccounts → get address → SIWE sign → POST /api/auth/callback/credentials
```
After that, NextAuth session JWT handles identity. No reconnect on navigation.

#### 3. Sign Transactions Without wagmi

For operations that need signing (donate, claim, profile save, crossword):
```ts
const tx = await window.ethereum.request({
  method: 'eth_sendTransaction',
  params: [{ to, value, data }],
})
```

#### 4. What We Keep

| Keep | Why |
|------|-----|
| NextAuth + SIWE backend | Works perfectly, session persists across navigation |
| `billetera_usuario.token` | API auth, already solid |
| `authenticateUser()` | Already decoupled from wagmi |
| viem utilities | Address formatting, ABI encoding (lightweight, no state) |

#### 5. What We Remove

| Remove | Replaced By |
|--------|-------------|
| `@rainbow-me/rainbowkit` | Own Connect button |
| `@rainbow-me/rainbowkit-siwe-next-auth` | Direct SIWE flow |
| `wagmi` (useAccount, useConnect) | `window.ethereum` + NextAuth |
| `@rainbow-me/rainbowkit/wallets` | Direct `eth_requestAccounts` |

### Benefits

| Benefit | Impact |
|---------|--------|
| **No reconnect popups** | wagmi state loss is irrelevant — we don't use wagmi for auth |
| **Single source of truth** | `session.address` for identity everywhere |
| **Smaller bundle** | Remove ~200KB (RainbowKit + wagmi dependencies) |
| **Control** | We own every line of auth/connect logic — bugs are ours to fix |
| **PWA-friendly** | No RainbowKit modal that breaks in standalone mode |
| **Prepares for R-#184** | Own connect flow is the foundation for the built-in wallet |

### Migration Path

1. **Phase 1:** Create own `<ConnectWalletButton />` that uses `window.ethereum`
2. **Phase 2:** Replace RainbowKit `ConnectButton` in Header
3. **Phase 3:** Replace wagmi `useAccount` in write components (donate, claim, profile save) with direct `window.ethereum` calls
4. **Phase 4:** Remove RainbowKit + wagmi packages
5. **Phase 5:** Write unit/E2E tests for the new flow

### Risk Mitigation

- **Mobile wallets (OKX, OneKey):** `window.ethereum` is the standard injected provider — all Web3 wallets inject it
- **WalletConnect:** Keep `@walletconnect/ethereum-provider` for QR-based connections (if needed)
- **SIWE compatibility:** Sign with `personal_sign` via `window.ethereum` — same as current flow

## Dependencies

- Keep: `next-auth`, `siwe`, `viem`
- Remove: `@rainbow-me/rainbowkit`, `@rainbow-me/rainbowkit-siwe-next-auth`, `wagmi`

## Acceptance Criteria

- [ ] User sees own Connect button (not RainbowKit's) in Header
- [ ] Clicking Connect opens wallet once — no repeated popups on navigation
- [ ] After SIWE, `session.address` persists across all page navigations
- [ ] Donate, claim UBI, profile save, crossword submit all work with direct `window.ethereum` calls
- [ ] No "Accept connection" popups on course pages
- [ ] Bundle size reduced (verify with `pnpm build` analysis)
- [ ] All existing E2E tests pass (or adapted to new flow)
