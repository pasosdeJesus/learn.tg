## Phase 2: Auto-Reconnect (Pendiente)

**Problem:** `useAuthAddress` falls back to session when wagmi disconnects, but
the user still sees "Connect Wallet" in RainbowKit's ConnectButton. On pages
where both wagmi and session are available, we should attempt to reconnect
wagmi automatically.

**Proposal:** Add a `useEffect` inside `useAuthAddress` that calls
`wagmi`'s `reconnect()` (or `connect({ connector: injected() })`) when
`session.address` exists but wagmi reports disconnected.

```ts
// Inside useAuthAddress:
useEffect(() => {
  if (sessionAddress && !wagmiAddress && !isConnected) {
    // Attempt reconnection silently
    connectAsync?.({ connector: injected() }).catch(() => {})
  }
}, [sessionAddress, wagmiAddress, isConnected])
```

This attacks the root cause: after navigation, wagmi's context is stale but the
injected provider is still available â€” a reconnect should succeed.

**Risk:** Infinite reconnect loops if the provider is genuinely unavailable.
Mitigate with a retry limit (3 attempts, 2s apart).
