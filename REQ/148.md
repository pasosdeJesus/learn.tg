Enable SLEARN as a payment method for premium courses, implement burning (`burn`) of SLEARN upon payment, integrate automated referral rewards, enable SLEARN ↔ Leone (SLE) swapping (stable-sl), and add church distribution improvements.

## ⏳ Dependencies
- Issue #129 completed (SLEARN and LearnTGVaultsV3 on mainnet, initial mint done).

## 🚧 Pending tasks

### 1. Premium Course Payments with SLEARN

- [ ] Modify backend so that when a user buys a premium course:
  - User approves SLEARN spending (or signs a transfer).
  - Backend calls `SLEARN.burn(address user, uint256 amount)` to burn the SLEARN paid.
  - `SLEARN` emits `Burn` event for auditing.
- [ ] Optional: allow mixed payments (partial USDT, partial SLEARN).

### 2. Burn Functionality in SLEARN Contract

- [ ] Verify backend has `BURNER_ROLE`.
- [ ] Implement `burnFrom` function (if using allowance) for more flexibility.

### 3. Automated Referral Program

- [ ] Ensure `SLEARN.receive` already handles referral distribution (design done, needs backend to pass `referralAddress` correctly).
- [ ] Create contract or script to distribute USDT accumulated in `referralWallet` to referrers (based on on-chain events).
- [ ] Display referral stats and pending rewards on user profile.

### 4. SLEARN ↔ Leone (SLE) Swap (stable-sl)

- [ ] Design `StableSLBridge` contract (or use multisig wallet).
- [ ] Implement swap logic: user sends SLEARN to bridge, bridge burns SLEARN and pays Leone (SLE) via Orange Money (off-chain initially).
- [ ] Add swap option in learn.tg UI (only for Sierra Leone users).

### 5. Church Distribution Improvements (Optional)

- [ ] If decided, implement a separate `ChurchesFund` contract that:
  - Receives USDT and SLEARN from `churchesWallet`.
  - Distributes to active churches based on metrics (number of scholarships given, referred students, etc.).
  - Allows pastors to register their wallets and verify activity.

### 6. Advanced Transparency Dashboard

- [ ] Show breakdown of SLEARN burned from premium payments.
- [ ] Display historical reserve and coverage ratio.
- [ ] Add distribution charts (to churches, referrals, UBI, etc.).

### 7. Governance (Phase 4 of Whitepaper)

- [ ] Design two‑chamber system (Learner House and Donor House).
- [ ] Implement governance contracts to adjust parameters (percentages, rates, authorizedTransfers) without redeploy.

## Notes
- Tasks in this issue can be implemented gradually after the initial launch.
- Not all are required for SLEARN v1.0; prioritize based on community feedback.
