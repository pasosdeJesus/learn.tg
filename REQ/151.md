## ✅ Done

### Sivel.xyz endpoint deactivation

- [x] `app/api/learning-points/increment/route.ts` — deactivated (returns 410 Gone)

### Scholarship migration: learningpoints → slearn

- [x] `lib/scores.ts` — `updateUserAndCoursePoints()` no longer inserts a fake transaction. The real SLEARN + USDT payment is recorded by `check-crossword/route.ts` via `payScholarship()` on-chain with `crypto='usdt'` + `crypto='slearn'` and actual amounts.
- [x] `lib/scores.ts:24` — `refreshUserLearningScore()` filters `crypto='slearn'` (was `'learningpoints'`)
- [x] Scores tests pass (3/3)
- [x] `check-crossword/route.ts` already used `crypto='usdt'` + `crypto='slearn'` — no changes needed

### Identity verification for partners

- [x] `app/api/verify/route.ts` — dual verification: name + nationality against Self.xyz passport
  - Compares `passport_name === nombre` **and** `passport_nationality === pais_id`
  - Returns `{ verified, name, countryId }` for authorized partners (sivel.xyz)
  - If user changes their name or country → loses verification automatically
  - EIP-191 signature with 5 min window, `AUTHORIZED_VERIFIERS` configurable
  - **Decision:** API as living source of truth (not SBT) — sensitive data (name, country) should not go on-chain and verification must be real-time
  - Tests pass (13/13)
