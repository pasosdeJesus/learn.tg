# R-#167: Fix Session Persistence Across Page Navigation

RainbowKit + SIWE reconnects the wallet on every page navigation, causing session loss, unnecessary wallet prompts, and poor UX — especially on mobile and PWA in Sierra Leone.

## Current Problem

```
User navigates /en/course1 → /en/course2
        │
        ▼
RainbowKit auto-reconnects (wagmi reconnectOnMount)
        │
        ▼
SIWE sign-in triggered → new nonce → billetera_usuario.token overwritten
        │
        ▼
NextAuth session may not pick up new token in time
        │
        ▼
User sees "not authenticated" or gets stuck re-signing
```

## Solution

Three targeted config changes — no new infrastructure, no new tables.

### 1. Disable Auto-Reconnect on Mount

RainbowKit's `wagmi` connector reconnects on every page mount. Disable it.

```typescript
// providers/Web3Provider.tsx or wagmi config
const config = createConfig({
  // ...
  connectors: [...],
})

// Disable auto-reconnect on mount
// wagmi v2: set reconnectOnMount: false in the connector config
```

### 2. Defer SIWE Verification

SIWE should only trigger when the user explicitly goes to `/profile` or initiates a wallet-requiring action (donate, pay). Not on every navigation.

```typescript
// app/api/auth/auth-options.ts
const authOptions = {
  providers: [
    CredentialsProvider({
      // ...
      authorize: async (credentials) => {
        // Only verify SIWE message when explicitly requested
        // (profile page, donate modal, claim UBI)
      },
    }),
  ],
  callbacks: {
    session: async ({ session, token }) => {
      // session.address already set from previous SIWE
      // Don't re-verify unless token is expired
    },
  },
}
```

### 3. Cache Session with Activity-Based Expiry

Store session with activity-based expiry — not a fixed 24h timer. Session stays alive as long as the user is active.

```typescript
// lib/session-cache.ts
const SESSION_KEY = 'learntg_session'
const INACTIVITY_TIMEOUT = 7 * 24 * 60 * 60 * 1000 // 7 days

export function cacheSession(address: string, token: string) {
  localStorage.setItem(SESSION_KEY, JSON.stringify({ 
    address, 
    token, 
    lastActive: Date.now() 
  }))
}

export function getCachedSession(): { address: string; token: string } | null {
  const raw = localStorage.getItem(SESSION_KEY)
  if (!raw) return null
  const data = JSON.parse(raw)
  // Expire only after 7 days of inactivity
  if (Date.now() - data.lastActive > INACTIVITY_TIMEOUT) return null
  return data
}

export function touchSession() {
  const raw = localStorage.getItem(SESSION_KEY)
  if (!raw) return
  const data = JSON.parse(raw)
  data.lastActive = Date.now()
  localStorage.setItem(SESSION_KEY, JSON.stringify(data))
}
```

Call `touchSession()` on every page navigation or user interaction. Session expires only after 7 days without activity — not on a fixed clock.

**Why 7 days:** Usability for Sierra Leone (limited connectivity, re-signing is a barrier). Security is maintained — financial operations still require wallet signature. Explicit logout still available.

## Acceptance Criteria

- [ ] Navigating between course pages does NOT trigger wallet reconnection
- [ ] Navigating between language pages does NOT trigger SIWE sign-in
- [ ] SIWE only triggers when user visits `/profile` or clicks "Donate"
- [ ] Session persists while user is active (up to 7 days without interaction)
- [ ] `touchSession()` called on every page navigation and user interaction
- [ ] Explicit logout clears session immediately
- [ ] Cache restores session on page refresh without wallet prompt
- [ ] Works in PWA standalone mode
- [ ] Works in mobile browsers (Chrome Android, Safari iOS)
- [ ] Existing SIWE flow still works for initial wallet connection
