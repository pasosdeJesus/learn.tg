# E-#177: Two-Layer Strategy: Celo as Operational Layer, Cardano as Resilience Layer

## Objective

Define a strategic roadmap for the pdJ ecosystem (learn.tg, SLEARN token, smart contracts, and identity system) that keeps Celo as the operational layer for the Global Disciples course and community, while evaluating and preparing Cardano as a future resilience layer for offline transactions, post-quantum security, and censorship-resistant communications.

## Background

The current ecosystem is built on Celo (Ethereum L2). It is functional and operational, with:

- SLEARN token (ERC-20) deployed and working
- LearnTGVaults and PasosDeJesusCredentials contracts deployed and working
- USDT + SLEARN scholarships working
- Global Disciples course designed and ready for launch on Celo

**The course is tied to Celo.** Migrating it now would:
- Delay the launch by 6-12 months
- Require rewriting all contracts in Plutus
- Confuse the community in Sierra Leone who already use Celo-based wallets
- Lose the traction already built on Celo

**However, Celo has limitations for the long-term vision:**

| Limitation | Cardano Advantage |
| :--- | :--- |
| **Offline capability:** Celo lacks native support for offline transactions | Cardano (via Hydra) has a planned "offline mode" for its L2 |
| **Post-quantum readiness:** Current system uses secp256k1 | Cardano's research into hash-based signatures (e.g., SPHINCS+) is more advanced |
| **Censorship resistance:** Celo's governance is more centralized | Cardano's governance model and node distribution may offer greater long-term resilience |
| **Ecosystem alignment:** Celo's focus is on stablecoins and mobile payments | Cardano's ecosystem includes Atala PRISM (identity) and Lorachain integration for offline mesh networks |

### Layer 1: Celo (Operational — Now)

**Purpose:** Run the Global Disciples course, manage scholarships, distribute SLEARN, and build the community in Sierra Leone.

| Component | Status | Action |
| :--- | :--- | :--- |
| SLEARN token | ✅ Deployed and working | Continue operations |
| LearnTGVaults | ✅ Deployed and working | Continue operations |
| Credentials Contract | ✅ Deployed and working | Continue operations |
| Global Disciples Course | 🔄 Ready for launch | **Launch on Celo** |
| Clusters + Ranking | 🔄 In development | **Complete on Celo** |
| Fund for Clusters | 🔄 In development | **Complete on Celo** |

**Decision:** The Global Disciples course and all operational features will be delivered on Celo. No migration of the core product.

### Layer 2: Cardano (Resilience — Future)

**Purpose:** Provide a parallel layer for offline transactions, post-quantum security, and communications that can operate without internet.

| Component | Status | Action |
| :--- | :--- | :--- |
| Offline Transactions | ❌ Not started | Research Hydra offline mode + Lorachain |
| Post-Quantum Identity | 🔄 Designed (#176) | Implement on Cardano |
| Communications Layer | ❌ Not started | Research encrypted messaging over mesh networks |
| Long-Term Asset Storage | ❌ Not started | Consider storing SLEARN reserves on Cardano |

**Decision:** Explore Cardano in parallel, without disrupting the Celo-based operations.

### The Bridge: Unified Identity

The user's identity is the bridge between layers:

1.  **One seed phrase** (SPHINCS+ based, from #176) derives:
    - A secp256k1 keypair for Celo (wallet for daily operations)
    - An Ed25519 keypair for Cardano (wallet for resilience layer)
2.  **Same user**, two wallets, one seed phrase.
3.  **No migration**: Users keep using Celo for the course. Cardano is an opt-in layer for advanced features.

## Scope

### 1. Technical Assessment (Cardano Layer)

Evaluate the effort required to build the Cardano resilience layer:

| Component | Cardano Implementation | Complexity | Notes |
| :--- | :--- | :--- | :--- |
| **Identity (#176)** | SPHINCS+ keypair derived from seed | **Medium** | Already designed; adapt to Cardano |
| **Offline Transactions** | Hydra offline mode + Lorachain | **High** | Research phase |
| **Communications** | Encrypted messaging over mesh | **High** | Future phase |
| **Asset Storage** | Cardano Native Tokens | **Medium** | For long-term reserves |

**Important:** This is not a migration. This is a **parallel build** that does not affect the Celo-based operations.

### 2. Key Questions for the Cardano Layer

- How can we leverage Hydra's offline mode for scholarship payments?
- Can we integrate Lorachain for offline transaction relay?
- How do we implement the SPHINCS+ identity registration on Cardano?
- What is the cost of running a Cardano node on OpenBSD/adJ?
- What grants are available for building the resilience layer (Project Catalyst, etc.)?

### 3. Strategic Alignment

| Aspect | Celo (Layer 1) | Cardano (Layer 2) |
| :--- | :--- | :--- |
| **Primary use** | Daily operations, course, scholarships | Resilience, offline, post-quantum |
| **Timeline** | Now | Post-MVP |
| **Community** | Pastors, students in SL | Same community, opt-in |
| **Grants** | Limited | Project Catalyst, UNICEF, etc. |

### 4. Implementation Roadmap

| Phase | Action | Duration | Dependencies |
| :--- | :--- | :--- | :--- |
