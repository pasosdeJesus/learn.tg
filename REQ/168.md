# R-#168: RainbowKit QR code crash when connecting mobile wallet from desktop

Clicking "Connect Wallet" → OKX Wallet (or WalletConnect) on Chrome desktop
crashes the RainbowKit modal immediately with "Connection Error".

---

## Diagnosis

### Symptom
- Click "Connect Wallet" → choose OKX Wallet
- RainbowKit modal crashes before showing QR code
- ErrorBoundary shows "Connection Error"
- No `siwe_auth_attempt` entries in DB → error occurs **before** authentication

### Root cause (confirmed with dev build)
```
AppProvider.tsx:113 Error: invalid border=0
    at encodeQR (index.js:1577:15)
    at Module.create (QrCode.js:9:62)
    at eval (Cuer.js:53:60)
    at Root (Cuer.js:48:59)
```

The **`cuer` v0.0.3** library (RainbowKit's internal dependency for rendering QR
codes) crashes when generating the QR code with `border=0`.

RainbowKit 2.2.11 (latest available version) depends on `cuer@0.0.3`. There is
no newer version of `cuer` published on npm.

### Full stack trace
```
Root (Cuer.js:48) → useMemo → QrCode.create → encodeQR (index.js:1577)
  ↑                              ↑                    ↑
React component             Creates QR instance    Crashes with
in RainbowKit modal         with options           "invalid border=0"
```

### Ruled out
- ❌ Not Tailwind CSS v4 — `@source` directive didn't help
- ❌ Not WalletConnect projectId — it's valid
- ❌ Not `@pasosdejesus/m` — error is in `cuer` (RainbowKit dependency)
- ❌ Not hostname/HTTPS/SIWE — never reaches that stage

## Next steps

- [x] Leer código fuente de `cuer` v0.0.3 para entender por qué `border=0` crashea
  - **Causa confirmada**: `cuer` usa `border: 0` al llamar `encodeQR()`, pero su
    dependencia transitiva `qr@0.6.0` (resuelta vía `"qr": "~0"`) agregó
    validación estricta que rechaza `border <= 0`.
  - Issue upstream: https://github.com/wevm/cuer/issues/12
- [x] Aplicar workaround: `pnpm.overrides` en `package.json` para forzar `qr@0.5.5`
- [x] Verificar en producción — conexión OKX Wallet desde Chrome desktop funciona
