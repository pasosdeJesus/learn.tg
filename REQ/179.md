## Test Propuesto: Full User Flow

Cubre el flujo que un usuario real hace:

```
1. Abrir learn.tg → ver Connect Wallet
2. Click Connect → OneKey aprueba → firma SIWE → recarga → dirección visible
3. Ir a /en → ver cursos con progreso y bóveda
4. Entrar a curso → ver progreso, bóveda, botón Donar
5. Donar → diálogo con balances → confirmar en wallet → toast de éxito
6. Ir a perfil → ver datos, cambiar display name, guardar
7. Ir a guía → ver contenido
8. Ir a crucigrama → llenar respuestas → submit → feedback
9. Ir a guía UBI → click Claim → ver diálogo
10. Desconectar ✕ → dirección desaparece → botón Connect Wallet
```

### Bloqueos a resolver

1. **Página `/en` no hidrata** — muestra solo CSS después del reload post-SIWE.
   Posible causa: wagmi `WagmiProvider` en `AppProvider` intenta inicializar con
   el mock y falla silenciosamente. Solución: migrar `AppProvider` a no depender
   de wagmi (Phase 4 de R-#186), o hacer que el mock sea compatible con wagmi.

2. **Crossword submit** — usa `useWriteContract` de wagmi. Solución: Phase 3 de R-#186
   (reemplazar con `window.ethereum` directo).

3. **Mock más completo con ethereumjs** — para simular `eth_sendTransaction`,
   `eth_call`, eventos. Permitiría testear donaciones, claims y crucigramas
   sin depender de contratos reales en Sepolia. Ver `siwe-wallet-mock.mjs` como
   punto de partida.
