## Test Propuesto: Full User Flow

Cubre el flujo que un usuario real hace:

```
1. Abrir learn.tg → ver Connect Wallet
2. Click Connect → OneKey aprueba → firma SIWE → recarga → dirección visible
3. Ir a /en → ver cursos con progreso y bóveda
4. Entrar a curso → ver progreso, bóveda, botón Donar
5. Donar → diálogo con balances → confirmar en wallet → toast de éxito
6. Ir a perfil → ver datos, cambiar display name, guardar
7. Ir a guía → ver contenido
8. Ir a crucigrama → llenar respuestas → submit → feedback
9. Ir a guía UBI → click Claim → ver diálogo
10. Desconectar ✕ → dirección desaparece → botón Connect Wallet
```

## Recomendaciones (2026-07-14)

### 1. Migrar `siwe-wallet-mock.mjs` → `@pasosdejesus/m/e2e`

El mock actual de learn.tg es superior al `simulateSIWE` del paquete `m`:
- `simulateSIWE` hace fetch directo al backend (no prueba el flujo UI real)
- `siwe-wallet-mock.mjs` con `exposeFunction` + firma ECDSA real + `evaluateOnNewDocument`
  permite que el frontend complete TODO el flujo: click botón → sign → reload

**Acción:** Exportar `setupSIWEMock` desde `@pasosdejesus/m/e2e` para que
cualquier proyecto pdJ (sivel.xyz, stable-sl) pueda reusarlo.

```js
// Uso futuro desde @pasosdejesus/m/e2e:
import { setupSIWEMock } from '@pasosdejesus/m/e2e'
const page = await browser.newPage()
await setupSIWEMock(page, address, privateKey, chainId)
```

### 2. Mock `eth_sendTransaction` con ethereumjs

Para testear donaciones y crucigramas sin depender de Sepolia:
- Simular `eth_sendTransaction` → devolver hash realista + eventos
- Simular `eth_call` para balances de tokens (CELO, cUSD, SLEARN)
- El mock actual devuelve `0xcd...cd` fijo — suficiente para Connect, no para Donate/Crossword

### 3. Test incremental (no big bang)

No esperar los 10 pasos. Implementar por bloques independientes:
- **Bloque A** (pasos 1-3): Connect + navegación → ya funciona
- **Bloque B** (pasos 4-5): Donate → requiere mock eth_sendTransaction
- **Bloque C** (paso 6): Perfil → requiere sesión, ya testeable
- **Bloque D** (paso 7): Guía → ya funciona
- **Bloque E** (paso 8): Crucigrama → requiere mock eth_sendTransaction
- **Bloque F** (pasos 9-10): UBI + Disconnect → ya funcionan

### 4. Smoke test HTTP-only

Validar que todos los endpoints del journey responden sin necesidad de Chrome:
- `GET /api/auth/csrf` → 200 + csrfToken
- `POST /api/auth/callback/credentials` → 302 (redirect)
- `GET /en` → 200 + contenido de cursos
- `GET /api/profile` → 200 (autenticado) o 401 (sin sesión)
- `GET /api/check-crossword` → 200

Esto corre en CI sin Puppeteer y detecta regresiones de backend rápido.
