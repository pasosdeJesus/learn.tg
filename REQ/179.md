## Bug Corregido: Token de Autenticación

**Problema:** `ConnectWalletButton` guardaba la dirección en localStorage pero
no el token de autenticación. `getCsrfToken()` en páginas subsecuentes retorna
un nonce diferente al almacenado en `billetera_usuario.token`, causando 401
en `/api/profile` y otros endpoints.

**Fix:**
- `components/ConnectWalletButton.tsx` — Guarda `learn.tg.authToken` en localStorage
- `app/[lang]/profile/page.tsx` — Lee token de localStorage, fallback a `getCsrfToken()`
- `components/CeloUbiButton.tsx` — Mismo fix

```js
// ConnectWalletButton — después de SIWE exitoso:
localStorage.setItem('learn.tg.sessionAddress', checksummedAddress)
localStorage.setItem('learn.tg.authToken', csrfToken)  // ← nuevo

// Páginas que llaman APIs:
const csrfToken = localStorage.getItem('learn.tg.authToken') || await getCsrfToken()
```

## Migración a @pasosdejesus/m/e2e

`setupSIWEMock` fue migrado de `e2e/helpers/siwe-wallet-mock.mjs` (local)
a `@pasosdejesus/m/e2e` como export oficial:

```js
// src/e2e/siwe-mock.ts → dist/e2e/siwe-mock.js
export async function setupSIWEMock(
  page: E2EPage,
  address: string,
  privateKey: string,
  chainId?: number,       // default: 42220
  opts?: SIWEMockOptions  // { fakeBalance, fakeCelo, fakeTxHash }
): Promise<void>
```

Disponible para cualquier proyecto pdJ: `import { setupSIWEMock } from '@pasosdejesus/m/e2e'`
