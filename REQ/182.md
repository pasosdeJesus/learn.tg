# R-#182: Centralized File Upload System (MSIP Pattern)

## Objective
Replace ad-hoc file upload paths with a centralized upload system following the MSIP pattern, where all attachments are managed by a single engine module. Files are organized by year and document type.

## Motivation
Currently, uploads use per-module subdirectories (`uploads/user/`, `uploads/church/`). MSIP manages all attachments centrally under a single directory with a unified naming convention, making backups, cleanup, and access control simpler.

## Proposal

### Directory structure

```
{NEXT_PUBLIC_UPLOAD_PATH}/
  {year}/
    id_photo_{userId}_front.{ext}
    id_photo_{userId}_back.{ext}
    church_registration_{churchId}.{ext}
    credential_{tokenId}.{ext}
    course_logo_{courseId}.{ext}
    guide_image_{guideId}.{ext}
```

### Rules

| Rule | Description |
|------|-------------|
| **Year-based** | Files are grouped by upload year, preventing any single directory from growing too large |
| **Flat per year** | One directory per year, no subdirectories by type. The filename encodes the type |
| **Plain IDs** | Filenames use the entity ID (user, church, etc.) plus type prefix and optional suffix |
| **Single engine** | One module (`lib/upload.ts`) handles all upload/serve/delete operations |
| **API-based serve** | All files served via API (not `public/`), with auth where needed |
| **Env var** | `NEXT_PUBLIC_UPLOAD_PATH=uploads` (base path, no trailing slash) |

### API endpoints (centralized)

| Endpoint | Method | Auth | Description |
|----------|--------|------|-------------|
| `/api/upload` | POST | Yes | Upload a file to `{year}/` |
| `/api/upload/{year}/{filename}` | GET | Varies | Serve a file |
| `/api/upload/{year}/{filename}` | DELETE | Yes | Delete a file |

### Current paths → New paths

```
user/123/id_front.jpg       → 2026/id_photo_123_front.jpg
user/123/id_back.jpg        → 2026/id_photo_123_back.jpg
user/456/registration.jpg   → 2026/church_registration_456.jpg
```

### Implementation

| Phase | What |
|-------|------|
| 1 | `lib/upload.ts` with `uploadFile()`, `deleteFile()`, `serveFile()` |
| 2 | `POST/GET/DELETE /api/upload` using the module |
| 3 | Migrate existing upload endpoints to use the new system |
| 4 | Migrate existing files from `uploads/user/` to the new structure |

## Dependencies
- R-#152 (church photos, ID photos, registration documents)
- Existing `NEXT_PUBLIC_UPLOAD_PATH` env var

## Out of Scope
- CDN/cloud storage (local filesystem only in MVP)
- Image resizing/thumbnailing
- Virus scanning
