# R-#188: Allow users to schedule interview for verification in the availability of the verifier and registering it with CalDav

## Objective

Allow users who have completed their profile to request a verification interview. The system shows the verifier's available time slots (based on a CalDAV calendar), lets the user select one, and creates a calendar event in the verifier's CalDAV calendar.

## Motivation

Currently, users can request verification, but there is no formal mechanism to schedule the interview. The verifier (pdJ staff) manually coordinates via WhatsApp/Telegram, which is inefficient and doesn't scale. Additionally, the verifier wants to keep their personal calendar private and not depend on Google Calendar or third-party services like Calendly.

### Components

| Component | Technology | Purpose |
|-----------|------------|---------|
| **Radicale** | CalDAV server | Store calendar events (busy slots, verification appointments) |
| **Calcurse** | CLI client | Verifier manages their calendar from terminal |
| **DAVx⁵** | Android app | Verifier manages calendar from mobile |
| **learn.tg API** | Next.js API routes | Show availability, book appointments |
| **Frontend** | React/Next.js | User selects time slot |

### Calendar Data Flow

1. **Verifier** manages their calendar via Calcurse (terminal) or DAVx⁵ (Android)
2. **Radicale** stores all calendar events
3. **learn.tg** queries Radicale to get busy slots
4. **learn.tg** calculates available slots based on:
   - Working hours (configurable)
   - Busy events (from Radicale)
   - Vacation/blocked days
5. **User** selects an available slot
6. **learn.tg** creates an event in Radicale (marking the slot as busy)

#### `POST /api/verification/book`

Books an appointment for the user.

**Request:**
```typescript
POST /api/verification/book
{
  "usuarioId": 123,
  "start": "2026-07-20T09:00:00Z",
  "end": "2026-07-20T09:30:00Z",
  "notes": "Optional user notes"
}
```

**Response:**
```json
{
  "success": true,
  "appointmentId": "uuid",
  "start": "2026-07-20T09:00:00Z",
  "end": "2026-07-20T09:30:00Z"
}
```

**Race condition protection:** The endpoint must re-check slot availability immediately before creating the CalDAV event. If the slot was taken between the user's selection and the POST, return 409 Conflict.

### 5. Component: `VerificationScheduler`

```tsx
// components/VerificationScheduler.tsx
'use client'

import { useState, useEffect } from 'react'
import { useSession } from 'next-auth/react'
import { useAuthAddress } from '@/lib/hooks/useAuthAddress'
import { useToast } from '@pasosdejesus/m/shadcn-components/ui/use-toast'
import { Button } from '@pasosdejesus/m/shadcn-components/ui/button'
import { createComponentT } from '@/lib/hooks/useTranslation'

interface Slot {
  start: string
  end: string
}

interface Props {
  lang?: string
  onBooked?: () => void
}

export function VerificationScheduler({ lang = 'en', onBooked }: Props) {
  const [slots, setSlots] = useState<Slot[]>([])
  const [selectedSlot, setSelectedSlot] = useState<Slot | null>(null)
  const [isLoading, setIsLoading] = useState(false)
  const [isBooking, setIsBooking] = useState(false)
  const { toast } = useToast()
  const { data: session } = useSession()
  const { address } = useAuthAddress()

  const t = createComponentT(lang, {
    en: {
      title: 'Schedule Verification Interview',
      description: 'Select a time slot for your verification interview. The interview takes about 20-30 minutes.',
      loadingSlots: 'Loading available slots...',
      noSlots: 'No slots available in the next 14 days. Please check back later.',
      book: 'Book Interview',
      booking: 'Booking...',
      success: 'Interview scheduled successfully!',
      error: 'Failed to schedule interview',
      unauthorized: 'Please connect your wallet first',
    },
    es: {
      title: 'Agendar Entrevista de Verificación',
      description: 'Selecciona un horario para tu entrevista de verificación. La entrevista dura entre 20 y 30 minutos.',
      loadingSlots: 'Cargando horarios disponibles...',
      noSlots: 'No hay horarios disponibles en los próximos 14 días. Por favor revisa más tarde.',
      book: 'Agendar Entrevista',
      booking: 'Agendando...',
      success: '¡Entrevista agendada exitosamente!',
      error: 'Error al agendar entrevista',
      unauthorized: 'Por favor conecta tu billetera primero',
    },
  })

  // Fetch available slots on mount
  useEffect(() => {
    if (session?.address && address) {
      fetchSlots()
    }
  }, [session, address])

  const fetchSlots = async () => {
    setIsLoading(true)
    try {
      const res = await fetch('/api/verification/availability?days=14&duration=30')
      const data = await res.json()
      if (data.slots) {
        setSlots(data.slots)
      } else {
        setSlots([])
      }
    } catch {
      toast({ title: t('error'), variant: 'destructive' })
    } finally {
      setIsLoading(false)
    }
  }

  const handleBook = async () => {
    if (!selectedSlot) return
    if (!session?.address || !address) {
      toast({ title: t('unauthorized'), variant: 'destructive' })
      return
    }

    setIsBooking(true)
    try {
      const token = localStorage.getItem('learn.tg.authToken')
      const res = await fetch('/api/verification/book', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({
          walletAddress: address,
          token,
          start: selectedSlot.start,
          end: selectedSlot.end,
        }),
      })
      const data = await res.json()
      if (!res.ok) throw new Error(data.error || t('error'))
      toast({ title: t('success') })
      setSelectedSlot(null)
      onBooked?.()
      fetchSlots() // Refresh slots
    } catch {
      toast({ title: t('error'), variant: 'destructive' })
    } finally {
      setIsBooking(false)
    }
  }

  if (isLoading) {
    return <p className="text-gray-500">{t('loadingSlots')}</p>
  }

  if (slots.length === 0) {
    return <p className="text-gray-500">{t('noSlots')}</p>
  }

  return (
    <div className="space-y-4">
      <p className="text-sm text-gray-600">{t('description')}</p>
      <div className="grid grid-cols-2 md:grid-cols-3 gap-2">
        {slots.map((slot, i) => (
          <Button
            key={i}
            variant={selectedSlot?.start === slot.start ? 'default' : 'outline'}
            className="text-sm"
            onClick={() => setSelectedSlot(slot)}
          >
            {new Date(slot.start).toLocaleString(lang === 'es' ? 'es' : 'en', {
              weekday: 'short',
              month: 'short',
              day: 'numeric',
              hour: '2-digit',
              minute: '2-digit',
            })}
          </Button>
        ))}
      </div>
      {selectedSlot && (
        <Button onClick={handleBook} disabled={isBooking} className="w-full">
          {isBooking ? t('booking') : t('book')}
        </Button>
      )}
    </div>
  )
}
```

### 6. Server Library: `lib/caldav.ts`

Uses HTTP directo (fetch con verbos PROPFIND/REPORT/PUT), sin dependencia de `ts-caldav`.

```typescript
// lib/caldav.ts

const CALDAV_URL = process.env.CALDAV_URL || ''
const CALDAV_USER = process.env.CALDAV_USER || ''
const CALDAV_PASS = process.env.CALDAV_PASS || ''

function authHeader(): string {
  return 'Basic ' + Buffer.from(`${CALDAV_USER}:${CALDAV_PASS}`).toString('base64')
}

export async function getBusySlots(start: Date, end: Date): Promise<{ start: Date; end: Date }[]> {
  const body = `<?xml version="1.0" encoding="utf-8"?>
<C:calendar-query xmlns:C="urn:ietf:params:xml:ns:caldav">
  <C:prop><C:calendar-data/></C:prop>
  <C:filter>
    <C:comp-filter name="VCALENDAR">
      <C:comp-filter name="VEVENT">
        <C:time-range start="${start.toISOString()}" end="${end.toISOString()}"/>
      </C:comp-filter>
    </C:comp-filter>
  </C:filter>
</C:calendar-query>`
  const res = await fetch(CALDAV_URL, {
    method: 'REPORT',
    headers: { 'Authorization': authHeader(), 'Depth': '1', 'Content-Type': 'application/xml' },
    body,
  })
  // Parse XML response: extract UIDs from <href> elements (Radicale returns
  // filename-based UIDs: /path/UID.ics). The REPORT query returns hrefs, not
  // inline calendar-data with UID properties.
  const text = await res.text()
  const uids: string[] = []
  const hrefRegex = /<href>([^<]+)<\/href>/g
  let match
  while ((match = hrefRegex.exec(text)) !== null) {
    const uid = match[1].split('/').pop()?.replace('.ics', '')
    if (uid) uids.push(uid)
  }
  return uids.map(uid => {
    // TODO: fetch individual .ics file to get start/end times
    return { uid, start: new Date(), end: new Date() }
  })
}

export async function createEvent(summary: string, start: Date, end: Date, description?: string): Promise<void> {
  const uid = crypto.randomUUID()
  const ics = [
    'BEGIN:VCALENDAR',
    'VERSION:2.0',
    'PRODID:-//learn.tg//Verification//EN',
    'BEGIN:VEVENT',
    `UID:${uid}`,
    `DTSTART:${start.toISOString().replace(/[-:]/g, '').split('.')[0]}Z`,
    `DTEND:${end.toISOString().replace(/[-:]/g, '').split('.')[0]}Z`,
    `SUMMARY:${summary}`,
    description ? `DESCRIPTION:${description}` : '',
    'END:VEVENT',
    'END:VCALENDAR',
  ].filter(Boolean).join('\r\n')
  await fetch(`${CALDAV_URL}${uid}.ics`, {
    method: 'PUT',
    headers: { 'Authorization': authHeader(), 'Content-Type': 'text/calendar' },
    body: ics,
  })
}
```

## OpenBSD/adJ Implementation Notes

### CalDAV via HTTP directo

No se usa `ts-caldav` (constructor privado, incompatible con este entorno).
En su lugar se usa `fetch()` nativo con verbos CalDAV:

| Operación | Método HTTP | URL |
|-----------|-------------|-----|
| Listar eventos | `REPORT` | `CALDAV_URL` |
| Crear evento | `PUT` | `CALDAV_URL/{uid}.ics` |
| Eliminar evento | `DELETE` | `CALDAV_URL/{uid}.ics` |

### Extracción de UIDs

Radicale no devuelve `<UID>` dentro de `<calendar-data/>` en la respuesta REPORT.
Los UIDs se extraen de los `<href>` del XML:

```typescript
const hrefRegex = /<href>([^<]+)<\/href>/g
// href tiene formato /verifier/default/UID.ics
const uid = href.split('/').pop()?.replace('.ics', '')
```

Para obtener detalles (start/end/summary) de cada evento, hacer GET individual al `.ics`.

### Autenticación

```typescript
const auth = Buffer.from(`${CALDAV_USER}:${CALDAV_PASS}`).toString('base64')
headers['Authorization'] = `Basic ${auth}`
```

### XML namespaces requeridos

```xml
<!-- REPORT body para calendar-query -->
<C:calendar-query xmlns:C="urn:ietf:params:xml:ns:caldav">
  <C:prop><C:calendar-data/></C:prop>
</C:calendar-query>
```

### Formato iCalendar

Líneas separadas por `\r\n` (CRLF). Sin trailing newline antes de END:VCALENDAR.
El UID es asignado por el cliente y debe ser único (usar `crypto.randomUUID()`).

```
BEGIN:VCALENDAR\r\n
VERSION:2.0\r\n
PRODID:-//learn.tg//Verification//EN\r\n
BEGIN:VEVENT\r\n
UID:{uuid}\r\n
DTSTART:{formatted}\r\n
DTEND:{formatted}\r\n
SUMMARY:{text}\r\n
END:VEVENT\r\n
END:VCALENDAR
```

### Variables en apps/.env

```bash
CALDAV_URL=https://caldav.pasosdeJesus.org/radicale/verifier/default/
CALDAV_USER=verifier
CALDAV_PASS=********
```

Sin variable `CALDAV_CALENDAR` redundante: el calendario va en la URL.
