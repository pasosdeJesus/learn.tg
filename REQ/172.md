## Objective
End-to-end test using Puppeteer to verify wallet session behavior across page
navigation on learn.tg. Tested against production (`https://learn.tg`).

## Test File
`apps/nextjs/test/puppeteer/session-persistence.mjs`

## Findings

### R-#168 (RainbowKit QR crash)
Tested and verified: QR code rendering crashes due to `qr@0.6.0` breaking change.
Fixed via `pnpm.overrides: { "qr": "0.5.5" }`.

### R-#167 (reconnectOnMount)
Test confirms `reconnectOnMount={false}` prevents wagmi from reconnecting after
remount. However, this causes `useAccount().address` to be `undefined` after
ANY navigation where the component tree remounts, breaking profile/course pages.

### R-#173 (Partial login)
Diagnosed via test: session cookie persists, but `address` is `undefined` after
navigation with `reconnectOnMount={false}`. Root cause documented in R-#173.
