# R-#172: E2E Session Persistence Test with Puppeteer

## Objective
End-to-end test using Puppeteer to verify wallet session behavior across page
navigation on learn.tg. Tested against production (`https://learn.tg`).

## Test File
`apps/nextjs/test/puppeteer/session-persistence.mjs`

## Tech Stack
- `puppeteer-core` (not `puppeteer` — only core works on OpenBSD)
- `siwe` + `viem` for simulating wallet signatures
- OpenBSD: requires `--ozone-platform=headless` for Chrome 141

## Dependencies
`apps/nextjs/test/puppeteer/package.json`:
```json
{
  "name": "learn-tg-puppeteer-tests",
  "version": "1.0.0",
  "private": true,
  "type": "module",
  "dependencies": {
    "puppeteer-core": "^24.0.0"
  }
}
```
SIWE/viem resolved via `NODE_PATH=../../node_modules`.

## Execution
```bash
cd apps/nextjs/test/puppeteer
npm install
NODE_PATH=../../node_modules CHROME_PATH=/usr/local/bin/chrome IPDES=learn.tg node session-persistence.mjs
```

## Capabilities (2026-07-08)

### ✅ Working
- Launch Chrome in headless mode on OpenBSD (`--ozone-platform=headless`)
- Navigate to `https://learn.tg` (production)
- Simulate SIWE authentication: fetch nonce → sign message → POST to NextAuth callback
- Verify NextAuth session cookie is set after SIWE
- Navigate between pages: `/` → `/en` → `/en/freecoder`
- Capture browser console logs (`PARTIAL LOGIN`, `NEXTAUTH_URL`, etc.)
- Detect "Partial login" text on any page
- Mock `window.ethereum` provider via `page.evaluateOnNewDocument`
- Verify session cookie persists across hard navigations (NextAuth JWT survives)
- Fetch and verify user profile data (`/api/user/:id`) — profilescore, SLEARN, credentials
- Detect whether account is verified (profilescore ≥ 50)

### ⚠️ In Progress
- Simulate wagmi wallet connection (currently `address: false` after remount)
  - Sin `reconnectOnMount={false}`, wagmi no se puede mockear fácilmente en Puppeteer
  - El mock de `window.ethereum` se inyecta pero wagmi no lo detecta como conector válido
  - Alternativa: testear con `reconnectOnMount={false}` explícito en una build de test

### 🔧 OpenBSD Chrome Issues
- Chrome 141 may hang after multiple launches — clean `/tmp/puppeteer*` between runs
- Requires `--ozone-platform=headless` flag
- Requires `--disable-gpu --disable-software-rasterizer`
- `DISPLAY` variable can interfere — unset for headless mode

## Findings (actualizado 2026-07-08)

### R-#168 (RainbowKit QR crash)
Tested and verified: QR code rendering crashes due to `qr@0.6.0` breaking change.
Fixed via `pnpm.overrides: { "qr": "0.5.5" }`.

### R-#167 (reconnectOnMount)
- `reconnectOnMount={false}` se aplicó (`4e24d6f`) y se revirtió (`ed853c9`)
- Causaba `useAccount().address === undefined` tras navegación → "Partial login"
- Ver [R-#167](167.md) para la nueva solución propuesta (aflojar guard)

### R-#173 (Partial login)
- Causa raíz confirmada: [NextAuth bug #5719](https://github.com/nextauthjs/next-auth/discussions/5719) — `useSession()` retorna `'unauthenticated'` tras navegación cliente aunque la cookie de sesión existe
- El E2E test **no reproduce** el bug con SIWE simulado — solo ocurre con wallet real (OneKey)
- Ver [R-#173](173.md) para diagnóstico completo
