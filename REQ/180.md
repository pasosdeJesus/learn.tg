# R-#180: Navigation Header & Footer Redesign (Hamburger Menu + Legal Links)

## Objective
Replace the current single "Profile" button in the header with a mobile-friendly hamburger menu (visible only after wallet connection) that provides access to key platform sections. Update the existing footer to include Privacy Policy and Terms of Service links for legal compliance.

## Dependencies
- Existing i18n system (`createComponentT` + `lib/i18n/common.ts`)
- Existing pages: `/[lang]/profile`, `/[lang]/transparency`, `/[lang]/leaderboard`
- Privacy Policy page: `/[lang]/privacy-policy`
- Terms of Service page: `/[lang]/terms-of-service`
- Existing `components/Header.tsx` (modify — replace Profile button with hamburger)
- Existing `components/Footer.tsx` (modify — add legal links)
- Existing `components/Layout.tsx` (already includes Header + Footer)
- Existing RainbowKit `ConnectButton` (remains unchanged)
- R-#152 (church pages — `/[lang]/church/[id]` — individual church detail)

> **Note:** There is no `/church` directory/index page in R-#152. The hamburger menu links to individual church pages via the "Profile" route, not a directory. If a church directory is needed, it should be a separate REQ.

## MVP Checklist (Phase 1)

### 1. Hamburger Menu (Header — Visible Only When Wallet Connected)

The hamburger menu (☰) replaces the current "Profile" `<Button asChild><Link>`. It only renders when the wallet is connected and the session is valid.

**Menu items (flat i18n keys matching project conventions):**

| Order | Key | Label (en/es) | Link | Emoji |
|-------|-----|--------------|------|-------|
| 1 | `navProfile` | Profile / Perfil | `/[lang]/profile` | 👤 |
| 2 | `navLeaderboard` | Leaderboard / Tabla de Líderes | `/[lang]/leaderboard` | 🏆 |
| 3 | `navTransparency` | Transparency / Transparencia | `/[lang]/transparency` | 📊 |

**Behavior:**
- Menu is **only visible when wallet is connected** (`isConnected && address && session?.address && session.address.toLowerCase() === address.toLowerCase()` — same condition as the current Profile button)
- Menu closes when clicking outside (document click listener or blur)
- Menu closes after selecting an item
- Active item is highlighted based on current pathname

**Mobile/Desktop:**
- **Mobile:** Hamburger icon (☰) on the left side, menu opens as a dropdown below
- **Desktop:** Same hamburger behavior; no horizontal inline links

**Header layout when connected:** `[☰ Menu]` (left) + `[RainbowKit ConnectButton]` (right)

### 2. Footer Update (Legal Links)

Modify the existing `components/Footer.tsx` to add Privacy Policy and Terms of Service links at the bottom.

| Key | Label (en/es) | Link |
|-----|--------------|------|
| `privacyPolicy` | Privacy Policy / Política de Privacidad | `/[lang]/privacy-policy` |
| `termsOfService` | Terms of Service / Términos de Servicio | `/[lang]/terms-of-service` |

**Placement:** Below the existing social links section, in a new row.

**Note:** No copyright notice — course copyrights are handled per-course.

## Implementation Notes

| Aspect | Decision |
|--------|----------|
| **NavMenu component** | Inline in `Header.tsx` (no separate file needed for 3 items) |
| **Visibility guard** | Same condition as current Profile button: `isConnected && address && session?.address?.toLowerCase() === address?.toLowerCase()` |
| **State** | `useState(false)` for menu open/close |
| **Outside click** | `useEffect` with document `mousedown` listener |
| **Styling** | Tailwind CSS — dropdown with `absolute` positioning below the ☰ button |
| **Icons** | Emoji (no additional dependencies) |
| **i18n** | Flat keys in `common.ts` (navbar items) + local keys in `Header.tsx` (`title`, `menu`) |
| **Footer** | Add legal links row below existing social section |
