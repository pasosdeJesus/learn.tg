# R-#152: Profiles of Church / GD Cluster

Build the foundation for Global Disciples cluster formation by enabling users to declare their church/place of worship, register as pastors, and form clusters of 3+ churches.

## Dependencies
None (base issue)

---

## MVP Checklist (Phase 1)

### 1. Church Relationship Declaration (User Profile)

Add a field to the user profile: **Church relationship** (`church_relationship`):

| Value | Description |
|-------|-------------|
| `pastor` | Pastor of a church |
| `lider` | Leader/elder |
| `miembro` | Member |
| *(empty)* | No relationship declared |

Only users with `church_relationship = 'pastor'` can declare a church and create/join clusters.

### 2. Church/Place of Worship Declaration

A pastor declares their church from their profile page or a dedicated church page.

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| Geographic location (country/city) | Selector | ✅ Yes | |
| Church/center name | Autocomplete + open text | ✅ Yes | If not in list, creates new entry (curation in Phase 2) |
| Pastor name | Text | ✅ Yes | |
| Pastor WhatsApp | Text | ✅ Yes | |

**Reward:** +32 profile score upon completion.

### 3. Church Registration (For Pastors)

| Field | Type | Required | Notes |
|-------|------|----------|-------|
| Denomination | Text | ❌ No (Phase 2) | |
| Government registration | File upload | ❌ No (Phase 2) | |
| Delegate wallet | Text | ❌ No (Phase 2) | |

**Note:** These fields will be added in Phase 2.

### 4. Cluster Formation (Code-Based)

| Step | Description |
|------|-------------|
| **1. Create cluster** | Pastor creates a cluster from their church page → receives a 6-char alphanumeric code (e.g. `A3K9M2`) |
| **2. Share code** | Pastor shares the code with other pastors via WhatsApp, email, etc. |
| **3. Join cluster** | Other pastors enter the code on their church page → join the cluster |
| **4. Cluster formed** | When 3+ churches have joined, the cluster status changes to "Formed" |

**Rules:**
- A church can only belong to one cluster at a time. Joining a new cluster removes it from the previous one.
- No expiration on codes.
- No maximum member limit.
- Members can be added at any time after formation.
- All churches in a cluster must be from the same country.
- Cluster name must be unique within the same country (3-50 characters).
- Any pastor in the cluster can change the cluster name. Changes are logged in cluster history.
- Any pastor can click "Leave cluster" from their church page. Their church is removed from the cluster.
- When the last pastor leaves, the cluster is automatically dissolved.

### 5. Cluster Page

| Section | Content |
|---------|---------|
| **Cluster name** | Set by any pastor in the cluster |
| **Members** | List of churches in the cluster with join dates |
| **Status** | "Formed" / "In Progress" |
| **Join code** | `A3K9M2` — for inviting more churches |
| **Leave cluster** | Button for each pastor to remove their church |
| **Cluster history** | Log of name changes, joins, and leaves |

### 6. Church Page Cluster Display

If the church belongs to a cluster, display:
```
## Iglesia Centro de Paz
**Cluster:** Cluster Esperanza (View cluster)
```

### 7. Cluster Page Access

- **Entry point:** From church page
- **Visible to:** All users (public page)

## Implementation Notes

| Aspect | Decision |
|--------|----------|
| **Database** | Tables `iglesia`, `clustergd`, `iglesia_clustergd` (join) |
| **User field** | `usuario.church_relationship` ENUM('pastor','lider','miembro') |
| **Cluster code** | 6-char alphanumeric, stored in `clustergd.code` |
| **Encryption** | Sensitive data (WhatsApp) handled in #153 |
| **One cluster rule** | A church can belong to only one cluster; joining a new one removes from previous |
| **Country consistency** | All churches in a cluster must be from the same country |
| **Name uniqueness** | Cluster name must be unique within the same country (3-50 chars) |
| **No limits** | No expiration on codes, no max member count |
| **Dissolution** | Cluster auto-dissolves when last pastor leaves (no owner) |
