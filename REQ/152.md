# R-#152 Profiles of Church / GD Cluster

## Objective
Build the foundation for Global Disciples cluster formation by enabling users to declare their church/place of worship, register as pastors, and form clusters of 3+ churches.

## Dependencies
None (base issue)

### 3. Cluster Formation (Code-Based)

| Step | Description |
|------|-------------|
| **1. Create cluster** | Pastor creates a cluster from their church page → receives a 6-char alphanumeric code (e.g. `A3K9M2`) |
| **2. Share code** | Pastor shares the code with other pastors via WhatsApp, email, etc. |
| **3. Join cluster** | Other pastors enter the code on their church page → join the cluster |
| **4. Cluster formed** | When 3+ churches have joined, the cluster status changes to "Formed" |

**Rules:**
- A church can only belong to one cluster at a time. Joining a new cluster removes it from the previous one.
- A pastor can only create one cluster at a time. If the cluster is disbanded, the pastor may create another.
- No expiration on codes.
- No maximum member limit.
- Members can be added at any time after formation.
- All churches in a cluster must be from the same country. When a pastor attempts to join a cluster with a church from a different country, the system will reject it with the message: *"No puedes unirte a este clúster porque tu iglesia está en [país_iglesia] y el clúster es de [país_cluster]."*
- The cluster name must be unique within the same country.

### 4. Cluster Page

| Section | Content | Who can edit |
|---------|---------|--------------|
| **Cluster name** | Pastors can set a name; must be unique within the same country | Any pastor/delegate in the cluster |
| **Created by** | Name and date of the creator | View only |
| **Members** | List of churches in the cluster with join dates | View only |
| **Status** | "Formed" / "In Progress" | Auto-updated |
| **Join code** | `A3K9M2` | View only |
| **Disband cluster** | Button to disband the cluster | Only the creator |

**Name management:**
- Any pastor or delegate in the cluster can change the cluster name
- Name must be 3-50 characters
- Name must be unique within the same country (validation in backend)
- Changes are logged in the cluster history

**Disband rules:**
- Only the pastor who created the cluster can disband it
- Disbanding removes all members from the cluster
- The cluster is marked as 'disbanded' and cannot be rejoined
- The creator can create a new cluster after disbanding
- A confirmation dialog is required before disbanding

**Confirmation dialog:**
> *"¿Estás seguro de que quieres disolver este clúster? Todos los miembros serán removidos y el clúster no podrá ser reactivado."*

| **One cluster rule** | A church can belong to only one cluster; joining a new one removes from previous |
| **One cluster per pastor** | A pastor can only create one active cluster at a time |
| **Country consistency** | All churches in a cluster must be from the same country |
| **Name uniqueness** | Cluster name must be unique within the same country |
| **No limits** | No expiration on codes, no max member count |
