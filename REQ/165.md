## Current Script

**Location:** `apps/nextjs/scripts/create-course.sh`

**Usage:**
```bash
./scripts/create-course.sh "Global Disciples" 5 "global-disciples" 4 "1.0" "1.0" "source/global-disciples.svg"
```

**What it produces:**
```
resources/
  en/global-disciples/guide1.md  → Template with 5-pillar structure
  en/global-disciples/guide2.md
  en/global-disciples/guide3.md
  en/global-disciples/guide4.md
  es/discipulos-globales/guia1.md  → Spanish template
  es/discipulos-globales/guia2.md
  es/discipulos-globales/guia3.md
  es/discipulos-globales/guia4.md
```

Plus printed commands for vault, credential, and Rails admin steps.

## Next Phase: DB Record Creation

The Rails backend manages `cor1440_gen_proyectofinanciero` with a logical framework:
- Objectives (`objetivopf`)
- Results (`resultadopf`)
- Indicators (`indicadorpf`)
- Activities = guides (`actividadpf`)

To avoid breaking Rails validations, the script should either:
1. **Use the Rails API** — POST to `/admin/proyectosfinancieros` endpoint
2. **Generate SQL with framework** — insert records matching existing course patterns

The API approach is safer. Requires the script to authenticate with the Rails admin session.
