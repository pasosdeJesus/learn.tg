# R-#174: Replace `alert()` with `toast()` Across the Application

## Problem

The application currently uses `window.alert()` for error, success, and warning
notifications. `alert()` blocks the UI, has an ugly native look (especially on
mobile/PWA), and doesn't integrate with the design system. The project already
has `useToast` from shadcn/ui (`@pasosdejesus/m/shadcn-components/ui/use-toast`)
used successfully in profile for 2 cases — but there are ~26 `alert()` remaining.

## Solution

Replace all `alert()` with `toast()` from shadcn/ui:

- `toast({ title: '...' })` — success / info
- `toast({ title: '...', description: '...', variant: 'destructive' })` — error

## Replacement Pattern

```tsx
// Before
alert('Error message')

// After
import { useToast } from '@pasosdejesus/m/shadcn-components/ui/use-toast'
const { toast } = useToast()
toast({ title: 'Error message', variant: 'destructive' })
```
